# 完善生产部署与故障处理文档

- 日期：2026-07-21
- 类型：文档与运维流程

## 背景

本次云服务器更新过程中遇到了更新包路径错误、低内存服务器同时构建 API/Web 时 Next.js 长时间停在生产构建阶段、服务器重启后需要确认旧容器与新镜像状态等问题。停止 Caddy、Web、API，仅保留 PostgreSQL 后单独构建 Web，部署成功。

## 改动内容

重写根目录 `DEPLOY.md`，形成完整的生产部署和运维手册，包含：

1. Docker Compose 架构和需要保护的持久化数据；
2. 首次部署、生产 `.env` 配置和域名 HTTPS；
3. 更新包的白名单制作、检查和上传方式；
4. 更新前数据库、上传文件和镜像回滚备份；
5. 低内存服务器分阶段构建及资源监控流程；
6. Flyway 迁移、接口、页面和 HTTPS 验证；
7. 服务器重启后的恢复检查；
8. 预构建镜像部署方式；
9. 应用镜像、数据库和上传文件回滚流程；
10. 本次实际遇到的问题及对应解决办法；
11. 日常备份、常用运维命令和上线检查清单；
12. 2026-07-21 实际部署结果和关键资源数据记录。

## 安全注意事项

- 文档未记录任何服务器密码、数据库密码、管理员密码或 SSL 私钥；
- 明确禁止执行 `docker compose down -v` 和 `docker volume prune`；
- 更新包明确排除生产 `.env`、IDE 文件、构建缓存和备份；
- `compose.prod.yml` 被明确说明为覆盖文件，必须与主 Compose 文件组合使用。
