# 知识库加载性能与访问统计

日期：2026-07-21

## 背景

前端知识库中包含较大 Markdown 文档时，目录点击会因为 Next.js 链接预取、服务端 Markdown 解析和目录接口携带正文而变慢。后台访问统计也需要从简单的访问次数扩展到 PV、UV、UIP 及访问环境维度。

## 本次改动

### 知识库性能

- 知识库目录、卡片、面包屑和上一篇/下一篇链接统一关闭 Next.js 自动预取，避免同时预取多个大文档详情页。
- `/api/notes/tree` 和后台目录改用只返回目录元数据的 JPA 投影，不再为树节点读取完整正文。
- `getNote`、`getNoteTree` 使用 React `cache`，避免同一详情请求中元数据和页面渲染重复访问接口。
- Markdown 渲染组件改为客户端组件，避免服务端对大 Markdown 内容重复构建完整 React 树。
- 增加知识笔记路由的加载骨架，打开大文档时立即显示加载反馈，而不是看起来像点击无效。

### 访问统计

- 页面访问记录增加匿名访客哈希、匿名 IP 哈希、浏览器、设备、网络类型和地域字段。
- 新增 Flyway V5 数据库迁移，为访问记录补充维度字段和查询索引。
- 地域优先读取反向代理提供的国家/地区请求头；没有代理地域时回退到浏览器语言地区代码。
- 后台统计支持最近 7、30、90、180、365 天，并按北京时间生成连续的每日 PV、UV、UIP 时间序列，包含无访问日期的 0 值。
- 后台统计页增加 PV、UV、UIP 指标卡、趋势折线图、地域/浏览器/设备/网络分布和热门页面排行。
- 前端匿名访客 ID 使用 `localStorage` 持久化，并上报网络类型、语言和时区信息。

### 编码与兼容性

- 统计默认维度名称改为数据库 `CHR()` 表达式和 Java Unicode 转义，避免终端或构建环境编码转换造成问号乱码。

- 对历史记录中的 `?`、`??` 等编码脏值统一按“未知”聚合展示，避免旧数据继续污染后台分布图。

## 验证方式

- `mvn test -q`
- `npm run lint`
- `npm run build`
- `docker compose build api web`
- Docker 启动后检查 `/api/health`、`/api/notes/tree`、大笔记详情页和 `/admin/analytics`。

## 回滚说明

如需回滚，可恢复本次修改的代码并停止使用 V5 统计字段；原有 `page_views` 表字段仍然保留，不影响既有页面访问记录读取。
