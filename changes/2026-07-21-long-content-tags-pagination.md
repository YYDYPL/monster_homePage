# 长文档、Markdown 导入、标签筛选与博客分页

## 改动日期

2026-07-20

## 改动目的

修复文章和知识库笔记粘贴长文档时的保存冲突，提升 Markdown/HTML 内容编辑体验，并为博客增加可配置标签筛选和分页能力。

## 改动内容

- 将文章、笔记的摘要和标签存储字段扩展为 PostgreSQL `TEXT`，并新增 Flyway V4 迁移；新库初始化脚本同步使用 `TEXT`。
- 对文章、笔记输入内容进行统一清洗，移除 NUL 和非法控制字符，避免复制内容造成数据库保存异常。
- 文章和笔记 slug 冲突时自动追加数字后缀；标签去重并限制最多 10 个。
- Markdown 编辑器支持导入 `.md`/`.markdown` 文件、富文本/语雀 HTML 粘贴、代码块、颜色和高亮样式、图片上传与粘贴截图上传。
- Markdown 内容渲染支持安全的原始 HTML、语雀图片属性和代码高亮。
- 博客支持标签列表、标签筛选、分页和每页数量选择。
- 提升图片上传限制和异常提示，增加图片真实字节签名校验；同步 Docker、Spring Multipart 和 Caddy 配置。
- 修复 Docker 管理员账号密码环境变量名称，并保留项目根目录 `AGENT.md` 的改动文档约束。

## 验证结果

- Maven 测试通过。
- Web lint 通过，仅有项目已有的 `<img>` 优化警告，无 error。
- Web 生产构建通过。
- Docker Compose 配置检查、镜像重建和服务健康检查通过。
- 本地 `/api/health`、标签、分页以及长文档文章/笔记接口冒烟测试通过，临时数据已清理。

## 备注

- 本次只部署到本机 Docker Desktop，不操作云服务器，也不执行 Git push。
- V1 迁移恢复为已执行的原始版本，避免 Flyway 校验和本地数据库已记录 checksum 不一致；长文档扩展由 V4 迁移负责。
